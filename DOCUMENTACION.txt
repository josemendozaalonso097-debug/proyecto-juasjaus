             CBTIS 258 FINANCIEROS - DOCUMENTACIÓN OFICIAL TÉCNICA

Este documento detalla la arquitectura completa, el flujo lógico, y la 
estructura de código del Sistema de Servicios Financieros del CBTis 258.

------------------------------------------------------------------------
1. VISIÓN GENERAL DE LA ARQUITECTURA
------------------------------------------------------------------------
El sistema sigue la arquitectura Cliente-Servidor separada en dos carpetas 
principales: "backend" (API en Python) y "frontend" (Cliente Web en Vanilla JS).

A. BACKEND (FastAPI - Python)
   - Función: Gestiona la lógica de negocio, autenticación, conexión a 
     base de datos y validaciones de seguridad.
   - Base de Datos: SQLite (`cbtis258.db`), fácil de respaldar localmente.
   - Seguridad: Restringe accesos externos mediante CORS, con tokens 
     de sesión y logging de seguridad configurado.
   - Puertos: Se ejecuta en el puerto 8000 mediante `uvicorn`.

B. FRONTEND (Vanilla JS / HTML / CSS / PWA)
   - Función: Interfaz de usuario "Premium" moderna.
   - Tecnologías Clave:
     * HTML5 con Tailwind CSS (vía CDN) para estilos rápidos.
     * ES6 Modules (JavaScript Nativo) para estructurar el código sin Node.js.
     * Service Workers (`sw.js`) y Manifest para funcionar como PWA.
     * Librerías Externas: jsPDF (Recibos PDF), ZXing (Escáner de cámara), 
       JsBarcode (Generación de código de barras Oxxo).
   - Puertos: Se ejecuta en el puerto 5501 mediante `http.server`.

------------------------------------------------------------------------
2. ARQUITECTURA DEL FRONTEND (LÓGICA Y MÓDULOS)
------------------------------------------------------------------------
El Frontend está 100% modularizado usando `import`/`export` de ES6.

Estructura de Directorios Clave (`frontend/js/`):
- `pages/`: Controladores principales por cada vista HTML.
  * `login.js`: Transición, validación de login (Google y Normal) y alerts.
  * `principal.js`: Inicializa el dashboard, carga historial, modales y perfil.
  * `tienda.js`: Inicializa la tienda escolar, carritos de compra y PDFs.
- `components/`: Módulos de interfaz reutilizables.
  * `carrito.js`: Arreglo local que guarda los productos, calcula totales y renderiza el HTML del carrito.
  * `pago.js`: Maneja simulación de la pasarela de pago (Tarjeta o Oxxo) y uso de la cámara.
  * `productos.js`: Inyecta los catálogos en los modales, y gestiona semestres y tallas.
  * `modales.js`: Lógica para abrir/cerrar modales y enviar reportes/quejas (Orientación).
  * `papeleria.js`: Validación y visualización de subida de archivos (Max 5MB).
  * `perfil.js`: Carga la foto, la muestra en la UI y maneja cambios.
  * `sidebar.js`: Maneja la navegación lateral responsiva.
- `api/`: Comunicación directa con el Backend (FastAPI).
  * `auth.js`: Verificación periódica de la sesión (`check-session`), logins y logouts.
  * Funciones de fetch con headers de seguridad y Bearer Tokens.
- `utils/`: Utilidades genéricas.
  * `toast.js`: Sistema de notificaciones moderno flotantes que reemplazó a los viejos `alert()`.
  * `pdf.js`: Inyección y dibujado de Canvas para exportar los pagos a recibos PDF.
  * `storage.js`: Administra datos en `localStorage` pero prioriza la API en línea.
  * `pageTransition.js`: Animaciones tipo cortina roja al saltar entre páginas.

------------------------------------------------------------------------
3. FLUJO LÓGICO DEL SISTEMA (¿CÓMO FUNCIONA?)
------------------------------------------------------------------------

A. INICIO DE SESIÓN
1. El usuario entra a `http://localhost:5501` (que redirige a `login.html`).
2. Digita sus credenciales o usa Google. `auth.js` envía un request POST al Backend.
3. El Backend valida contra SQLite y devuelve un Token.
4. El Frontend guarda el Token y via `pageTransition.js` muestra una cortina visual y navega a `principal/index.html`.

B. FLUJO DE TIENDA Y COMPRA
1. En `tienda/tienda.html`, el usuario da clic en una categoría (ej. "Uniformes").
2. `productos.js` lee el catálogo en `api/tienda.js` y llena el HTML del modal correspondiente.
3. Al agregar al carrito, `carrito.js` añade el ítem al arreglo general de productos.
4. Al dar clic en "Pagar", `pago.js` despliega el simulador.
5. Si elige OXXO, se genera un código de barras. Si elige pagar de forma manual, y aprueba, se genera éxito.
6. A continuación, el sistema limpia el arreglo mediante `carrito.length = 0` (Para evitar bugs de caché y estado en memoria).
7. Se usa `pdf.js` para crear un recibo digital automático en el navegador del usuario y se manda a la base de datos (y perfil) de Historial.

------------------------------------------------------------------------
4. EJECUCIÓN DEL SISTEMA
------------------------------------------------------------------------
En Linux/macOS:
$ ./run.sh
Este script:
1. Mata puertos remanentes viejos.
2. Levanta Uvicorn en el puerto 8000 en segundo plano.
3. Levanta http.server en el puerto 5501.

En Windows:
Ejecuta el archivo `run.bat`

------------------------------------------------------------------------
5. DEPURACIÓN, CACHÉ Y CÓMO ESTÁ ESCRITO (NOTAS DE MANTENIMIENTO)
------------------------------------------------------------------------
- SISTEMA DE TOASTS: Totalmente prohibido usar `alert()` estándar. Al haber errores de validación (por ejemplo, archivo pesado) se debe llamar a `showToast(mensaje, 'error')`.
- ACTUALIZACIONES Y CACHÉ: Cuando se suben correcciones al JavaScript (ej. `.js`), asegúrate de agregar o incrementar su sufijo de versión en los tags `<script src="...?v=5">` y en los `import`. Al no usar NodeJS ni Webpack, el navegador (Chrome/Edge) guarda en memoria intensa los ES6 Modules. Bumping al parámetro de versión fuerza la recarga.
- ARTE Y DISEÑO: Se usan clases Tailwind. Para fondos usamos una animación de "mesh gradient" (CSS puro), tarjetas Glassmorphism (`backdrop-filter`) y sombras tenues.
